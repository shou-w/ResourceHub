RAGチャットボットの共有と独立セッション管理

1つのURLで共有しても、各ユーザーが独立したチャットセッションを持つ仕組み

システム構成

AIフォルダとベクトル化

AIフォルダ内のドキュメントをベクトル化してデータベースに格納します

  • 1 PDF、Word、テキストなどのファイルを保存
  • 2 埋め込みベクトルに変換
  • 3 ベクトルデータベースに格納

共有URL

1つのURLでAIチャットボットを共有します

https://example.com/ai-chat/finance-bot

このURLは特定のAIフォルダに紐づいたチャットボットを指します

セッション管理

各ユーザーごとに独立したセッションを作成します

  • 1 URLアクセス時に自動的にセッション生成
  • 2 セッションIDをブラウザに保存
  • 3 会話履歴をセッションごとに管理

テクニカル実装方法

フロントエンド側

1

セッションIDの生成と保存

初回アクセス時にランダムなセッションIDを生成し、localStorageに保存

const sessionId = crypto.randomUUID();
localStorage.setItem('chatSessionId', sessionId);
2

APIリクエスト時にセッションIDを送信

チャットメッセージ送信時にセッションIDを含める

fetch('/api/chat', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-Session-ID': localStorage.getItem('chatSessionId')
  },
  body: JSON.stringify({ message: userInput })
});
3

会話履歴の管理

ローカルで会話履歴を保持し表示

// メッセージ履歴を状態として保持
const [messages, setMessages] = useState([]);

// 新しいメッセージを追加
setMessages(prev => [...prev, 
  { role: 'user', content: userInput },
  { role: 'assistant', content: response.message }
]);

バックエンド側

1

セッション管理

セッションIDごとに会話コンテキストを保持

// セッションストア（メモリ、Redis、DBなど）
const sessions = new Map();

// リクエストからセッションIDを取得
const sessionId = req.headers['x-session-id'];

// セッションが存在しない場合は初期化
if (!sessions.has(sessionId)) {
  sessions.set(sessionId, { 
    messages: [],
    botId: req.params.botId
  });
}
2

RAG処理と応答生成

ユーザーの質問に基づいてベクトル検索と回答生成

// セッションからボットIDを取得
const { botId, messages } = sessions.get(sessionId);

// ベクトル検索の実行
const searchResults = await vectorDB.search(
  userMessage, 
  botId,
  { limit: 5 }
);

// AIモデルに送信するコンテキスト作成
const context = searchResults.map(r => r.text).join('\n');

// AI応答の生成
const aiResponse = await generateAIResponse(
  messages, 
  userMessage,
  context
);
3

会話履歴の更新

セッションごとのメッセージ履歴を更新

// セッションのメッセージ履歴を更新
const sessionData = sessions.get(sessionId);
sessionData.messages.push(
  { role: 'user', content: userMessage },
  { role: 'assistant', content: aiResponse }
);
sessions.set(sessionId, sessionData);

// クライアントに応答を返す
res.json({ message: aiResponse });

まとめ

  • 単一のURLで複数のユーザーにAIチャットボットを共有できます
  • 各ユーザーのチャットセッションは完全に独立しており、他のユーザーには見えません
  • 同じナレッジベース（AIフォルダのベクトル化されたファイル）を参照しながら、個別の会話コンテキストを維持します
  • フロントエンドまたはバックエンドでのセッション管理により、URLをシンプルに保ちながら個別の会話を実現できます
